ECE 472/572 Assignment 1

Purpose

Skills

The purpose of this assignment is to help you practice the following skills that are essential to your success in this course / in school / in this field / in professional life beyond school:

  • How to setup a virtual machine for an operating system and select its rudimentary parameters such as hard drive space, amount of RAM, and number of CPU cores.
  • How to choose appropriate version of and then install an operating system from the Debian Linux family of OS.
  • How to run commands in a command line interface (CLI).
  • How to install OS updates and software packages using a command line interface (CLI).

Knowledge

This assignment will also help you to become familiar with the following important content knowledge in this discipline:

  • What is a command line interface.
  • Basic set of commands that can be used in a command line interface.

Prepare for the lab

Answer these questions while doing this lab – they will help you understand the procedure and better prepare for the tests.

  • What is a virtual machine? What are the main benefits of virtual machines? Consider if you were using this at your job.
  • How does a virtual machine create space for a new operating system without partitioning your hard drive?
  • What are the main limiting factors on the number of virtual machines that can be running simultaneously?
  • What are the host operating system (host OS), and guest operating system (guest OS)? Can both be Windows, or both be Linux?
  • Can you share files between your host PC and a Virtual Machine? Think of a situation where this is not desirable.
  • Do you need a valid license to run an OS in a virtual installations?
  • What is grub2 loader? How does it differ from grub? What will this program allow you to do?
  • Protected files cannot be modified by users. How do you gain root access to a file, or open a program with root access?
  • What are apt and apt-get?
  • What is the difference between Ubuntu, Kubuntu, Xubuntu, Lubuntu, and Ubuntu Server?
    ( See also https://askubuntu.com/questions/690/what-is-the-difference-between-ubuntu-and-its-derivatives)
  • What is the difference between Ubuntu and Linux Mint?  Research at least a few communities to lear about different users' opinions.

Setting up Ubuntu inside a virtual machine

  1. Skip this step if you do your work in the computer lab as the VM is already installed:
    Download and install Oracle's Virtual Machine - Virtual Box from https://www.virtualbox.org/. It is a desktop application. Alternatively use VmWare's Virtual Machine - VMWare Player from https://www.vmware.com/ . Then donwload and install Virtual Box extension pack that contains non-open source componenets of Virtual Box that are necessary for efficient operation. VMWare is not open source so there is no extra step.
  2. The guest OS will be a lightweight distribution of Ubuntu 22.04.5 or 24.04.1, so start downloading now to have it handy for later step. If you are rookie to Linux, make sure that you download the desktop version and not the server version of Ubuntu to make your life easier at later time. /* It is OK to use an odd numbered major version or .10 minor version but the instructor will support only LTS versions */
    • https://cdimage.ubuntu.com/xubuntu/releases/24.04/release/xubuntu-24.04.1-desktop-amd64.iso or
    • https://cdimage.ubuntu.com/ and pick up your choice of K-, L-, or X-Ubuntu, or
    • https://linuxmint.com/download_all.php . Note that Mint 22.N matches Ubuntu 24.04.N, Mint 21.N matches Ubuntu 22.04.N, i.e., it is based on repackaged LTS version of Ubuntu.
  3. Mean while before your download is being completed start working on the virtual machine. Choose "Create a new Virtual Machine"
    1. "I will install OS later".
    2. "Linux" type of OS, Version "Debian64" or "Ubuntu64"
      • If you cannot choose 64-bit operating systems you need to restart your computer, run BIOS setup before starting your operating system, and enable virtualization support in the BIOS settings.
    3. Pick a name for your virtual machine, and remember the location so that you could later inspect the files supporting your machine.
    4. Hard drive you are recommended to use at least 32GB for the hard drive size.  This will be a file in your Windows system that will eventually grow to that size as needed. When the course is complete you can simply delete it. The rationale:
      • you need about 25GB total hard drive space for playing with Linux, and for the labs 1 through 4, and to do only the first parts of Labs 8 through 11, Lab 12 and end of the semester extra credit labs.
      • you need between 32GB and 40GB plan to complete Lab 5 (recompile Kernel for your OS)
      • you need at least 32GB to do Lab 6, and continue full development related to microcontroller Linux in the second parts of Lab 7 through 12 assuming you delete your Lab 5 work.
        If your computer is hard-drive-space-challenged then you could do these in the lab instead as you would eventually need to test your work on the embedded system there.
      • Your hard drive will not be expanded immediately (unless you request to do so for the sake of faster performance). It will grow as necessary.
      • Putting your virtual machine on an SD card or USB stick may be not the best idea but it is doable. Just be prepared to spend up to two days doing Lab 5 or Lab 6.
    5. Customize RAM memory Memory size is NOT the amount of RAM in your PC! This is the amount of RAM you want to dedicate to the Virtual installation. It is recommended to leave a minimum of 1GB remaining for the exclusive use of the host operating system (i.e. for Windows) and allow the guest system (i.e. Ubuntu Linux) to use the reminder if necessary.
      • You need at least 2GB of RAM to play with Linux and do all labs except Lab 6, 6, and 7.
      • We recommend that you assign 4GB of RAM to be able to do all labs at reasonable speed.
      • Unlike the case of hard drive space, you can always power off the virtual machine, and adjust the memory settings.
    6. Customize other hardware
      • Remove printer, floppy, e.t.c.
      • Consider "disconnecting" the real DVD drive (if you still have one), and "insert" only *.ISO files when needed.
    7. By the way, have your Ubuntu *.ISO file download that you started a while ago complete by the time you read this?
    8. YYou can always customize the virtual machine later as long as it is not running. It would be hard to change the size of the virtual hard drive though, especially to shrink it.
  4. Start the virtual machine. Start the Ubuntu install process. You can use the automatic install process as long as you do not choose "unattended installation."
  5. Reboot the virtual machine computer and log in into Ubuntu.
  6. In case of problems with screen resolution when running inside VirtualBox: (Similar solutions are available for VmWare.) ol>
  7. Open terminal window and run:
    sudo apt-get install virtualbox-guest-dkms virtualbox-guest-x11
  8. Restart the machine and test if the screen can be enlarged after logging in.
    In case the procedure above did not work:
    1. sudo apt-get purge virtualbox-*
    2. sudo apt-get install build-essential
    3. In the virtual box MENU → devices → Inser guest additions CD image
    4. cd /media/your_user_id/<hit TAB>
    5. sudo ./autorun.sh
    6. and after the entire operation is complete restart the virtual machine. (sudo reboot)
  • Update the OS. You will see a pop up box alerting you to probably just two updates available. If only that were all. After you complete the first updates, check again for updates. Select and update all. This will take a significant amount of time, and does not need to be attended. Find something to keep you busy for a while.
  • Under the application menu, select Add/Remove. Find the Gnome Partition Manager and install it. (This package is an application so it can be installed from the console, but you should explore the graphical interface as well.)
  • To clean up cached installation files open a terminal window and run:
    sudo apt-get autoremove
    sudo apt-get clean

    • Additional configuration that needs to be applied to the system

    Attention. In case your update fails due to an unrecognized package name, or package unavailable, repeat the command with that particular package name removed. Then inspect the spelling of the failed package name and try to install it separately. In case of error message stating that one (or more) package cannot be found, repeat the installation command without listing that package. This request applies to all future lab instructions as well until the end of the semester.
